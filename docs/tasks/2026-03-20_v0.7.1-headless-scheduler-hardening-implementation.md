## Registration steps

1. Capture pre-change scheduler state with `schtasks /Query /V /FO LIST` for:
   - `CivilizationRadar-WeekdaySnapshots`
   - `CivilizationRadar-FridayPromote`
   - `CivilizationRadar-MonthEndPipelineTag`
2. Run:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ops\register_weekly_tasks.ps1
```

3. Provide the current user password interactively when prompted.
   - This is a one-time registration cost.
   - It does not mean the tasks should keep depending on an active interactive session after registration.
4. If non-interactive registration fails partway, restore the prior registration mode with:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File .\ops\register_weekly_tasks.ps1 -InteractiveOnly
```
