# v0.6 Release Notes

## v0.6 PR Scope

This PR adds `global_baseline` as a new geo profile without changing v0.5 weighted profile behavior.

- New profile in `config/geo_profiles_v1.json`:
  - `global_baseline`
  - `kind: baseline`
  - `min_total: 200`
  - `min_countries: 6`
  - `cap_share: 0.18`
  - `alpha: 0.25`
- No manual country list is used for baseline.
- Baseline vector is derived from snapshot `top_countries` shares after capping and renormalization.
- Compatibility scalar for baseline:
  - `geo_factor = 1.0` when gate passes
  - `geo_factor = 0.0` when gate fails
### Semantics note: global_baseline geo_factor

For `global_baseline`, `geo_factor` is **not a boost signal**.
It is emitted as `1.0` solely to indicate that the baseline profile is active and gated.

Downstream consumers MUST NOT treat `geo_factor` from `kind=baseline`
as a multiplicative push. The primary artifacts of `global_baseline`
are `baseline_vector` and derived deltas.
baseline_vector ordering:
- primary: share descending
- secondary (tie-break): country code ascending

This preserves backward compatibility while keeping baseline semantics explicit.
